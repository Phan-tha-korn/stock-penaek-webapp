# คู่มือการใช้งานสำหรับ ADMIN

บทบาท `ADMIN` เน้นงานปฏิบัติการและดูแลข้อมูล แต่ไม่มีสิทธิ์อนุมัติงาน verification สำคัญ

## 1. เมนูที่ ADMIN เห็น

### เมนูหลัก
- `หน้าดูแลงานหลัก`
- `การแจ้งเตือน`

### ตรวจสอบงาน
- `เทียบสินค้าแบบไว`
- `คิวตรวจสอบอัตโนมัติ`

### ข้อมูลงาน
- `จัดการสินค้า`
- `จัดการร้านค้า`
- `ธุรกรรม`
- `รายงาน`

### การตั้งค่า
- `ตั้งค่าและหน้าตา`

## 2. ADMIN ใช้ระบบยังไงดีที่สุด

1. เปิด `หน้าดูแลงานหลัก`
2. เช็ก `การแจ้งเตือน`
3. ถ้ามีเรื่องต้องเตรียมข้อมูล ให้ไป `จัดการร้านค้า` หรือ `จัดการสินค้า`
4. ถ้าต้องดูผลเทียบ ให้เปิด `เทียบสินค้าแบบไว`
5. ถ้าต้องติดตามคิว ให้เปิด `คิวตรวจสอบอัตโนมัติ`

## 3. เมนูแต่ละตัวใช้ทำอะไร

### 3.1 หน้าดูแลงานหลัก
ใช้ดูสรุปงานปฏิบัติการของ ADMIN เช่น:
- งานที่กำลังค้าง
- งาน supplier ที่ต้องติดตาม
- สัญญาณผิดปกติของระบบ

### 3.2 เทียบสินค้าแบบไว
ใช้ดูราคาและเปรียบเทียบสินค้าแบบเร็ว

ADMIN ใช้เมนูนี้เมื่อ:
- ต้องตรวจว่าราคาที่กรอกไว้ดูสมเหตุสมผลไหม
- ต้องช่วยเตรียมข้อมูลให้ DEV/OWNER ตัดสินใจ
- ต้องเช็กว่าสินค้ากลุ่มเดียวกันราคาต่างกันมากไหม

### 3.3 คิวตรวจสอบอัตโนมัติ
ADMIN เปิดดูได้ แต่ไม่มีสิทธิ์อนุมัติ

ควรใช้เพื่อ:
- ติดตามว่างานไหนกำลังค้าง
- ดูว่ามีงานอะไรต้องเร่งประสาน
- เช็กว่าผลงานที่ส่งไปเข้า queue แล้วหรือยัง

### 3.4 จัดการสินค้า
ใช้ดูและแก้ข้อมูลสินค้าในส่วนที่บทบาท ADMIN ทำได้

### 3.5 จัดการร้านค้า
เป็นหน้าที่ ADMIN ใช้บ่อยที่สุด

ใช้ทำอะไร:
- เพิ่มร้านค้า
- แก้ข้อมูลร้านค้า
- ดูคะแนนความน่าเชื่อถือ
- ดูไฟล์แนบ
- ส่งข้อมูลเข้า flow ตรวจสอบ
- ลบร้านค้าออกจากรายการใช้งานแบบเก็บประวัติ

ข้อสำคัญ:
- ถ้าการเปลี่ยนแปลงนั้นเป็น protected change ระบบจะพาเข้า verification workflow
- ADMIN ดูได้ แต่ไม่ใช่คนอนุมัติสุดท้าย

### 3.6 ธุรกรรม
ใช้ตรวจ movement หรืออ้างอิงข้อมูลที่เกี่ยวข้องกับสินค้า/ร้านค้า

### 3.7 รายงาน
ใช้ดู summary และข้อมูลย้อนหลังในมุมงานปฏิบัติการ

### 3.8 การแจ้งเตือน
ใช้รับข่าวสารว่ามี:
- งานเข้าคิวแล้ว
- งานถูกส่งกลับแก้ไข
- งาน supplier ที่เกี่ยวข้อง

### 3.9 ตั้งค่าและหน้าตา
ใช้ปรับโหมดสี ภาษา และการแสดงผล

## 4. Flow ที่ ADMIN ใช้บ่อย

### เพิ่มหรือแก้ร้านค้า
1. เปิด `จัดการร้านค้า`
2. กรอกข้อมูล
3. แนบไฟล์ถ้าจำเป็น
4. บันทึก
5. ถ้าระบบมองว่าเป็น protected change งานจะเข้า verification queue
6. ADMIN ตามดูผลต่อที่ `คิวตรวจสอบอัตโนมัติ`

### ช่วยเตรียมข้อมูลราคาก่อนให้ DEV/OWNER ตรวจ
1. เปิด `เทียบสินค้าแบบไว`
2. เลือกสินค้าที่เกี่ยวข้อง
3. เช็กผลเทียบเบื้องต้น
4. ส่งต่อข้อมูลให้คนอนุมัติ

## 5. สิ่งที่ ADMIN ต้องระวัง

- อย่าคิดว่าดู queue ได้แปลว่าอนุมัติได้
- ถ้าร้านค้าหรือราคามีผลกระทบสำคัญ ให้ปล่อยเข้าระบบตรวจสอบตาม flow
- เวลาลบร้านค้า ให้เข้าใจว่าเป็น soft delete/เก็บประวัติ ไม่ใช่ลบถาวร
