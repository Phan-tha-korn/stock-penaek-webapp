# คู่มือการใช้งานสำหรับ OWNER

บทบาท `OWNER` คือผู้เห็นภาพรวมทั้งระบบ เห็นได้ทุก branch และมีสิทธิ์ตรวจสอบงานสำคัญเทียบเท่าหรือสูงกว่า Dev

## 1. เมนูที่ OWNER เห็น

### เมนูหลัก
- `ภาพรวมผู้บริหาร`
- `การแจ้งเตือน`

### ตรวจสอบงาน
- `เทียบสินค้าแบบไว`
- `คิวตรวจสอบอัตโนมัติ`
- `มุมมองเชิงลึก`

### ข้อมูลงาน
- `จัดการสินค้า`
- `จัดการร้านค้า`
- `ธุรกรรม`
- `รายงาน`

### ระบบ
- `จัดการผู้ใช้`
- `ตั้งค่าและหน้าตา`

## 2. OWNER ควรเริ่มใช้งานจากตรงไหน

ลำดับแนะนำ:

1. เปิด `ภาพรวมผู้บริหาร`
2. ดู `คิวตรวจสอบอัตโนมัติ`
3. เปิด `เทียบสินค้าแบบไว`
4. ถ้าต้องตรวจย้อนหลัง ให้เปิด `รายงาน` หรือ `มุมมองเชิงลึก`
5. ถ้ามีเหตุผิดปกติ ให้ตามต่อจาก `การแจ้งเตือน`

## 3. เมนูแต่ละตัวใช้ทำอะไร

### 3.1 ภาพรวมผู้บริหาร
ใช้ดูภาพรวมระบบในมุมบริหาร เช่น:
- จำนวนงานรอตรวจสอบ
- จำนวนงานเกินเวลา
- จำนวนราคาที่ใช้งานอยู่
- การเปลี่ยนแปลงสำคัญล่าสุด
- สัญญาณปัญหาจากการแจ้งเตือน

ใช้เมื่อ:
- เปิดระบบตอนเช้า
- ตรวจความพร้อมก่อนอนุมัติงาน
- ต้องการดูว่ามีเรื่องค้างหรือผิดปกติหรือไม่

### 3.2 เทียบสินค้าแบบไว
ใช้ค้นหาและเทียบสินค้าแบบเร็ว เหมาะกับการตัดสินใจหน้างาน

วิธีใช้:
1. เข้าเมนู `เทียบสินค้าแบบไว`
2. พิมพ์คำค้นหาได้เลย เช่น
   - ชื่อสินค้า
   - รหัสสินค้า
   - ชื่อร้านค้า
   - alias
3. ระบบจะแสดงรายการแนะนำอัตโนมัติ
4. กดเลือกสินค้าอย่างน้อย `2` รายการ
5. เลือกจำนวนที่ต้องการใช้เทียบ
6. ระบบจะสรุปให้ว่า:
   - ร้านไหนราคาดีสุด
   - ต้นทุนรวมเท่าไร
   - ราคา base เท่าไร
   - ใช้ delivery/area แบบไหน
7. ถ้าต้องการดูเชิงลึก กด `รายละเอียด`

เหมาะใช้เมื่อ:
- ต้องเลือกของที่ราคาดีสุดตอนนี้
- ต้องเทียบหลายสินค้าพร้อมกันแบบเร็ว
- ต้องเปิดจาก notification หรือ deep-link แล้วดูผลทันที

### 3.3 คิวตรวจสอบอัตโนมัติ
ใช้ดูงานที่ค้างตรวจสอบโดยไม่ต้องค้นหาก่อน

ระบบจะขึ้นให้อัตโนมัติ:
- งาน pending
- งานเกินเวลา
- งานที่ติด dependency

วิธีใช้:
1. เปิดเมนู `คิวตรวจสอบอัตโนมัติ`
2. ดูสรุปด้านบน:
   - รายการทั้งหมด
   - งานเกินเวลา
   - งานที่ติดบล็อก
3. ถ้าจะหาแบบคร่าว ๆ ให้พิมพ์ช่อง `ค้นหาแบบไว`
4. ถ้าจะหาเฉพาะเจาะจง ใช้:
   - เลขที่คำขอ
   - สถานะ
   - ระดับความเสี่ยง
   - ผู้รับผิดชอบ
5. เปิดรายการที่สนใจแล้วตัดสินใจ:
   - อนุมัติ
   - ปฏิเสธ
   - ส่งกลับแก้ไข
   - ยกระดับ

### 3.4 มุมมองเชิงลึก
ใช้ดูภาพเชิงลึกที่มากกว่าหน้าสรุป เช่น:
- ปัญหาที่ซ้ำบ่อย
- รูปแบบราคา
- งานที่กระทบหลายส่วน
- ภาพรวมที่ผู้บริหารใช้ตัดสินใจ

### 3.5 จัดการสินค้า
ใช้สร้าง แก้ไข ตรวจสอบ และดูรายละเอียดสินค้า

ใช้เช็ก:
- รหัสสินค้า
- ชื่อสินค้า
- หมวดหมู่
- ข้อมูลพื้นฐาน
- สถานะสินค้า
- รายละเอียดเชิงลึกจากหน้าสินค้า

### 3.6 จัดการร้านค้า
ใช้ดูและจัดการร้านค้า

ใช้ทำอะไรได้:
- สร้างร้านค้าใหม่
- แก้ไขข้อมูลร้านค้า
- ดูคะแนนความน่าเชื่อถือ
- ดูไฟล์แนบ
- ลบร้านค้าออกจากรายการใช้งานแบบเก็บประวัติ

### 3.7 ธุรกรรม
ใช้ดูการเคลื่อนไหวที่เกี่ยวข้องกับสินค้าและสต็อก

### 3.8 รายงาน
ใช้ดูข้อมูลสรุปและย้อนหลัง

OWNER ควรใช้รายงานเมื่อ:
- ต้องส่งข้อมูลให้ทีม
- ต้องดูข้อมูลย้อนหลังที่ stable
- ต้องอ้างอิง snapshot/report ที่ห้ามเปลี่ยนตาม live

### 3.9 การแจ้งเตือน
ใช้ดูเหตุการณ์สำคัญทั้งหมด

ตัวอย่างสิ่งที่เจอ:
- งานรอตรวจสอบใหม่
- งานเกินเวลา
- pricing change สำคัญ
- supplier change ที่ต้องตรวจสอบ

วิธีใช้:
1. เปิด notification
2. ดูหัวข้อและสถานะ
3. กดเปิดรายการที่เกี่ยวข้อง
4. ทำงานต่อจากหน้าปลายทาง

### 3.10 จัดการผู้ใช้
ใช้ตรวจสอบว่าผู้ใช้แต่ละคนมี role อะไร และเข้าถึงข้อมูลถูกต้องหรือไม่

### 3.11 ตั้งค่าและหน้าตา
ใช้ปรับ:
- ภาษา
- โหมดสี
- รูปแบบการแสดงผล
- ค่าพื้นฐานของระบบบางส่วน

## 4. งานที่ OWNER ควรตรวจทุกวัน

1. เปิด `ภาพรวมผู้บริหาร`
2. เปิด `คิวตรวจสอบอัตโนมัติ`
3. ดู `การแจ้งเตือน`
4. เปิด `เทียบสินค้าแบบไว` ถ้ามีงานจัดซื้อหรือเปรียบเทียบสินค้า
5. เปิด `รายงาน` ถ้าต้องสรุปย้อนหลัง

## 5. สิ่งที่ OWNER ต้องระวัง

- อย่าใช้ราคาแค่ตัวเลขเดียว ต้องดู `ต้นทุนรวมที่ใช้เทียบ`
- ถ้างานติด dependency อย่ารีบอนุมัติโดยไม่เปิดรายละเอียด
- ถ้างานมาจาก notification ควรตรวจหน้าปลายทางให้ครบก่อน
- เวลาใช้รายงานย้อนหลัง ให้ใช้ข้อมูลจาก snapshot/report ที่ตรึงไว้
